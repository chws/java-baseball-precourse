## 문제 정의
1~9의 서로 다른 수로 이루어진 3자리의 수를 맞추는 게임  
스트라이크: 값, 값의 인덱스가 일치할 때  
볼: 값은 일치하지만 인덱스가 일치하지 않을 때  
낫싱: 아무것도 일치하지 않을 때  
게임이 끝나는 조건: 3 스트라이크  

## 기능 목록
1. 게임 시작
2. 컴퓨터가 정답인 세자리 수 생성
3. 사용자 입력
4. 입력값과 정답을 비교하여 정답 여부 판단, 결과 전달  
    -스트라이크, 볼인지 확인하여 전달
5. 결과 출력
6. 게임 종료

## 개선할 점
- [x] 메소드 좀 더 추상화시키기(더 쪼개기)  
- [X] 메인함수 더 간결하게 만들기  
- [X] 예외처리 강화  
- [X] 변수처리  
    ex. `ball > 0` 에서 `0`을 변수로 바꿔서 이후에 변경하기 쉽게  
- [X] 이름 더 구체적으로 바꿔보자  
    ex. `option, arr` 이런 이름들
